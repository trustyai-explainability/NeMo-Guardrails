This guide shows you how to preview the Fern documentation site while editing MDX pages.

## Quick Start

Start the local Fern development server from the repository root:

```bash
make docs-fern-live
```

For a branch preview that publishes to Fern, run:

```bash
make docs-fern-preview-watch
```

## Prerequisites

Fern is run through the pinned CLI version in `fern/fern.config.json`. The commands use `npx` to fetch that version.

The Python SDK reference is generated automatically before the Fern checks and preview commands run.

## Available Methods

### Local Fern Server

```bash
make docs-fern-live
```

- Regenerates the Python SDK reference under `docs/_static/python-sdk-reference`.
- Normalizes the generated SDK tree to avoid duplicate sidebar entries.
- Starts the Fern local development server.

### Published Branch Preview

```bash
make docs-fern-preview-watch
```

This watches local docs and `fern/` configuration changes and publishes a Fern preview for the current Git branch.

### Check Only

```bash
make docs-fern-strict
```

This regenerates the SDK reference, normalizes it, and runs `fern check`.

### Direct Fern Command

```bash
make docs-fern-generate-sdk
cd fern
npx --yes fern-api@$(node -p "require('./fern.config.json').version") docs dev
```

## How It Works

1. **SDK Reference Generation**: Fern generates Python library reference pages from the `nemoguardrails` package.
2. **Normalization**: `scripts/normalize-fern-sdk-reference.mjs` removes duplicate generated sidebar entries.
3. **Fern Dev Server**: Fern starts a local documentation server.
4. **Live Reload**: Your browser refreshes as Fern rebuilds changed pages.

## What Files Are Watched?

The Fern workflow watches:

- MDX files in `docs/`
- Fern configuration in `fern/`
- Static assets referenced by the docs

Files ignored:

- Generated SDK reference output (`docs/_static/python-sdk-reference/`)
- Temporary files (`.swp`, `*~`)
- Git files (`.git/`)

## Accessing the Documentation

Use the local URL printed by Fern in the terminal.

## Stopping the Server

Press `Ctrl+C` in the terminal to stop the server.

## Troubleshooting

### Fern CLI Fails to Start

The Fern CLI is downloaded through `npx`. If it fails to start, check network access and rerun:

```bash
make docs-fern-live
```

### Changes Not Reflecting

Regenerate the SDK reference and restart the local server:

```bash
make docs-fern-generate-sdk
make docs-fern-live
```

### Browser Not Auto-Refreshing

- Make sure you're viewing the URL printed by Fern.
- Some browser extensions may block live reload.
- Try a different browser or incognito mode

## Tips

1. **Keep the terminal visible**: You'll see build progress and any errors
2. **Check for errors**: Red text in the terminal indicates build warnings or errors
3. **Generated files**: Do not edit files under `docs/_static/python-sdk-reference/`; regenerate them.
4. **Clean builds**: If things look wrong, stop the server and rerun `make docs-fern-live`.

## Advanced Configuration

The Make targets automatically:

- Use the pinned Fern CLI version.
- Regenerate and normalize Python SDK reference docs.
- Run Fern check or the Fern development server.

To customize, edit:

- `Makefile`
- `fern/docs.yml`

Or run Fern directly from the `fern/` directory:

```bash
npx --yes fern-api@$(node -p "require('./fern.config.json').version") docs dev
```
